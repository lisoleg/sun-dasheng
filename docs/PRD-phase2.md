## 1. Phase 2 产品目标

### 1.1 业务目标
1. **专业级交易终端体验**：将 MVP 形态的 UI 重构为面向专业量化交易员的"类 Bloomberg Terminal / TradingView Pro"工作台，提升日均使用时长与信号采纳率
2. **闭环策略验证能力**：在投入真金白银前，通过完整回测引擎对鲁兆理论各子模块进行历史验证与参数寻优
3. **多账户/多市场协同**：支持跨 A 股+币安双市场的统一账户视图与回测对比，构建"一个工作台管理两类资产"的核心壁垒

### 1.2 体验目标
- **首屏信息密度**：1.6K 屏可同时展示 6+ 个核心指标（行情、信号、盈亏、风险、持仓、回测进度）
- **关键操作 ≤3 次点击**：从"看到信号"到"查看置信度/推理过程/历史回测"不超过 3 次点击
- **零刷新体验**：通过 WebSocket 实时推送价格、信号、风控状态，避免手动刷新

### 1.3 范围边界（Non-Goals）
- ❌ 移动端 App（Phase 3 再考虑，本期仅保证 Web 端 1366×768 起步可用）
- ❌ 社区策略市场（Phase 3）
- ❌ A 股自动下单（受合规约束，本期仅完成回测验证）
- ❌ 多语言国际化（仅中文）

## 4. UI 设计描述

### 4.1 整体设计语言

| 维度 | 规范 |
|------|------|
| **设计风格** | 专业量化交易终端（深色主题为主，参考 TradingView Pro / Bloomberg Terminal / Eagle Eye） |
| **主色板（深色）** | 背景 `#0d1117` / 面板 `#161b22` / 边框 `#30363d` / 主文 `#c9d1d9` / 次文 `#8b949e` |
| **主色板（浅色）** | 背景 `#ffffff` / 面板 `#f6f8fa` / 边框 `#d0d7de` / 主文 `#1f2328` / 次文 `#656d76` |
| **语义色** | 涨 `#ef4444`（A 股红涨）/ 跌 `#22c55e`（A 股绿跌）/ 警示 `#f59e0b` / 危险 `#dc2626` / 信息 `#3b82f6` |
| **字体** | 数字 `JetBrains Mono` / `Roboto Mono`（等宽，便于对齐）；正文 `Inter` / `PingFang SC`（中文） |
| **数字对齐** | 金融数字一律右对齐 + 等宽字体；金额保留 2 位小数；百分比 2 位小数；大额自动 K/M 单位 |
| **图标库** | `lucide-react`（开源、清晰、风格统一） |
| **间距** | 8px 基准栅格（4/8/12/16/24/32/48） |
| **圆角** | 4px（按钮/输入框）/ 6px（卡片）/ 8px（弹窗） |
| **阴影** | 深色主题下使用 0 0 0 1px `#30363d` 模拟边框；浅色主题使用 `0 1px 3px rgba(0,0,0,.08)` |

## 5. 回测引擎功能规格

### 5.1 系统定位

回测引擎是 Phase 2 后端核心模块，用于**离线**验证鲁兆理论各子模块在历史数据上的表现，输出可量化的绩效指标，作为实盘参数配置的决策依据。

> ⚠️ **重要**：回测结果不构成投资建议；过去的表现不代表未来收益。

### 5.4 算法流程

#### 5.4.1 整体流程

```
┌─────────────┐
│ 1. 数据加载  │ → 从数据库/Provider 加载历史 K 线
└──────┬──────┘
       ↓
┌─────────────┐
│ 2. 信号计算  │ → 对每根 K 线运行启用的理论引擎
└──────┬──────┘    输出每根 K 线的理论信号
       ↓
┌─────────────┐
│ 3. 信号融合  │ → 根据 fusion 策略聚合多理论信号
└──────┬──────┘
       ↓
┌─────────────┐
│ 4. 事件循环  │ → 按时间顺序遍历 K 线，触发开仓/平仓事件
└──────┬──────┘
       ↓
┌─────────────┐
│ 5. 指标计算  │ → 计算收益、回撤、夏普等绩效指标
└──────┬──────┘
       ↓
┌─────────────┐
│ 6. 报告生成  │ → 组装 BacktestResult，写入数据库
└─────────────┘
```

#### 5.4.2 信号计算

- 对每根 K 线（按时间正序）：
  1. 调用 `TaijiEngine.analyze(bars_so_far)` → 返回太极中心点 + DNA 时间窗信号
  2. 调用 `SpiralEngine.analyze(bars_so_far)` → 返回斐波那契回撤位信号
  3. 调用 `BgMovingAverageEngine.analyze(bars_so_far)` → 返回金叉/死叉/底部形态信号
  4. 其他启用的理论模块...
- 每个引擎输出统一格式：
  ```python
  TheoryResult(
      theory_name="taiji",
      timestamp=bar.timestamp,
      signal=SignalHint(direction=BUY|SELL|HOLD, confidence=0.0~1.0),
      annotations={...}  # 用于图表标注
  )
  ```

#### 5.4.3 信号融合

- **AND 模式**：仅当所有启用的理论都给出相同方向信号时才下单（保守）
- **OR 模式**：任一理论给出信号即下单（激进）
- **加权模式**：按配置的权重计算加权置信度，超过阈值（默认 0.5）才下单

#### 5.4.4 事件循环（核心）

```python
def run_event_loop(bars: List[Bar], config: BacktestConfig) -> BacktestState:
    state = BacktestState(initial_capital=config.initial_capital)
    
    for i, bar in enumerate(bars):
        # 1. 更新当前持仓市值
        state.update_position_value(bar.close)
        
        # 2. 检查止损止盈（用当前 bar 的最高/最低价模拟）
        for pos in state.open_positions:
            if bar.low <= pos.stop_loss_price:
                state.close_position(pos, pos.stop_loss_price, 'STOP_LOSS', bar.timestamp)
            elif bar.high >= pos.take_profit_price:
                state.close_position(pos, pos.take_profit_price, 'TAKE_PROFIT', bar.timestamp)
            else:
                # 追踪止损
                new_stop = max(pos.stop_loss_price, 
                               bar.close * (1 - config.trailing_stop_pct))
                pos.stop_loss_price = new_stop
        
        # 3. 计算当前 bar 的理论信号
        theory_signals = run_theories(bars[:i+1], config.enabled_theories)
        fused_signal = fuse_signals(theory_signals, config.signal_fusion, config.theory_weights)
        
        # 4. 处理融合信号
        if fused_signal.direction == 'BUY' and not state.has_position(bar.symbol):
            # 计算仓位
            size = calculate_position_size(state, fused_signal, config)
            # 模拟滑点和手续费
            entry_price = bar.close * (1 + config.slippage_rate)
            # 开仓
            state.open_position(bar.symbol, size, entry_price, fused_signal, bar.timestamp)
        
        elif fused_signal.direction == 'SELL' and state.has_position(bar.symbol):
            # 平仓
            exit_price = bar.close * (1 - config.slippage_rate)
            state.close_position(state.get_position(bar.symbol), exit_price, 'SIGNAL', bar.timestamp)
        
        # 5. 记录权益曲线
        state.record_equity(bar.timestamp)
    
    return state
```

#### 5.4.5 关键算法

| 算法 | 公式 | 说明 |
|------|------|------|
| **夏普比率** | `(R_p - R_f) / σ_p × √252` | R_p=策略收益, R_f=无风险利率(0), σ_p=策略波动率 |
| **索提诺比率** | `(R_p - R_f) / σ_downside × √252` | 只用下行波动率 |
| **最大回撤** | `max((peak - trough) / peak)` | 滚动峰值与谷值 |
| **卡尔玛比率** | `年化收益 / |最大回撤|` |  |
| **盈亏比** | `平均盈利 / |平均亏损|` |  |
| **VaR (95%)** | 历史模拟法第 5 百分位 |  |
| **胜率** | `盈利交易数 / 总交易数` |  |
