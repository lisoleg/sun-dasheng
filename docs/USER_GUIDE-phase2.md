## 1. 系统简介

孙大圣 v0.2.0 是一个专业级量化交易系统，融合：

- **鲁兆理论**（7 个子模块：太极中心律 / 螺旋律 / 波浪理论 / 对偶律 / 周期律 / 江恩角度线 / BG 均线）
- **TOMAS-AGI**（翻译官 EML 快速检索 + 作家 LLM 创造性推理）
- **事件驱动回测引擎**（完整实现，支持参数扫描 + 并行优化）
- **专业交易终端 UI**（Bloomberg 风格深色主题 + 可拖拽面板）

支持市场：**A 股**（通达信接口）+ **币安**（REST + WebSocket API）

## 2. 安装部署

### 2.1 环境要求

| 软件 | 最低版本 | 推荐版本 |
|------|----------|----------|
| Python | 3.11 | 3.13 |
| Node.js | 18 | 22 |
| Redis | 5.0 | 7.2 |
| npm | 9 | 10 |

### 2.2 后端部署

```bash
cd backend
python -m venv venv
venv\Scripts\activate          # Windows
source venv/bin/activate    # Linux/Mac
pip install -r requirements.txt
```

配置 `.env`：

```bash
# 必填
DEEPSEEK_API_KEY=sk-xxxxxxxx     # TOMAS 作家引擎
BINANCE_API_KEY=xxxxxxxx      # 币安交易（Testnet 先跑）
BINANCE_API_SECRET=xxxxxxxx

# 选填（有默认值）
BINANCE_TESTNET=true
BACKTEST_MAX_WORKERS=4
LOG_LEVEL=INFO
```

初始化数据库：

```bash
alembic upgrade head   # 运行迁移（含 backtest_runs 表）
```

启动服务（3 个终端）：

```bash
# 终端 1：FastAPI
uvicorn app.main:app --reload --port 8000

# 终端 2：Celery Worker
celery -A app.tasks.celery_app worker --loglevel=info

# 终端 3：Celery Beat（定时行情采集）
celery -A app.tasks.celery_app beat --loglevel=info
```

### 2.3 前端部署

```bash
cd frontend
npm install
npm run dev
```

打开 `http://localhost:5173`。

> **注意**：首次加载会请求 `http://localhost:8000` 的 API，请确保后端已启动且 CORS 已配置。

## 3. 首次配置

### 3.1 连接交易所

1. 进入 **设置页**（`/settings`）
2. 点击 **「交易所 API」** Tab
3. 填入币安 API Key + Secret
4. 勾选「使用 Testnet」（推荐首次测试）
5. 点击 **「测试连接」** 验证
6. A 股行情无需 Key，填入通达信服务器 IP 即可（默认 `120.76.192.85`）

### 3.2 配置策略参数

1. 在设置页点击 **「策略参数」** Tab
2. 展开每个理论模块，调整参数：
   - **太极中心律**：DNA 窗口类型（DNA29 / DNA13）
   - **螺旋律**：斐波那契回撤位（0.236 / 0.382 / 0.5 / 0.618 / 0.786）
   - **波浪理论**：识别敏感度（1-5）
   - **对偶律**：连续对偶次数阈值
   - **周期律**：周期长度范围
   - **江恩角度线**：角度线类型（1×1 / 2×1 / 1×2）
   - **BG 均线**：均线周期（4 / 8 / 16 / 32）
3. 点击 **「保存到偏好」**

### 3.3 主题选择

- 点击右上角 **🌙/☀️ 图标** 切换深色/浅色主题
- 或进入设置页 **「系统」** Tab 选择布局模板（默认 / 极简 / 分析师）

## 8. 回测中心

> **回测引擎是 Phase 2 的核心新功能**，支持完整的历史策略验证。

### 8.1 配置回测

进入 **回测中心**（`/backtest`），填写配置表单：

#### 基础配置

| 字段 | 说明 | 示例 |
|------|------|------|
| 标的 | 交易对（输入自动补全） | `BTCUSDT` / `000001.SZ` |
| 市场类型 | A 股 / 币安 | `binance` |
| 时间周期 | K 线周期 | `1d`（日线） |
| 开始日期 | 回测起始日 | `2021-01-01` |
| 结束日期 | 回测结束日 | `2026-06-01` |
| 初始资金 | 初始账户资金 | `10000`（USDT） |

#### 交易参数

| 字段 | 说明 | 默认 |
|------|------|------|
| 手续费率 | 每笔交易的手续费比例 | `0.001`（0.1%） |
| 滑点（bps） | 基点滑点（0.1% = 10bps） | `5`（0.05%） |
| 止损百分比 | 固定止损（-1 = 不启用） | `0.05`（5%） |
| 止盈百分比 | 固定止盈（-1 = 不启用） | `0.10`（10%） |
| 追踪止损百分比 | 追踪止损（-1 = 不启用） | `0.03`（3%） |
| 最大仓位百分比 | 单笔最大仓位 | `0.3`（30%） |

#### 策略配置

| 字段 | 说明 | 选项 |
|------|------|------|
| 启用理论 | 选择参与信号生成的理论 | 7 个可独立勾选 |
| 理论权重 | 各理论的权重（WIGHTED 模式） | 滑块（0 - 1） |
| 信号融合策略 | 多理论冲突时的融合方式 | `AND` / `OR` / `WEIGHTED` |
| 仓位管理 | 仓位计算方式 | `fixed_pct` / `fixed_amount` / `kelly` / `risk_parity` |
| 允许做空 | 是否允许做空（A 股建议关闭） | 开关 |
| 盈利再投资 | 盈利是否加入下一笔仓位 | 开关 |

### 8.2 启动回测

1. 填写完配置后，点击 **「启动回测」** 按钮
2. 系统返回 `run_id`（回测唯一标识）
3. 进度条实时显示（通过 WebSocket 推送）：
   ```
   [==========          ] 45%  |  运行中...  |  当前阶段：回放 K 线 732/1500
   ```
4. 回测完成后进度条消失，显示结果面板

## 9. 风控监控

### 9.1 实时风险仪表

4 个仪表盘（SVG 或 recharts RadialBarChart）：

| 仪表 | 说明 | 阈值（默认） |
|------|------|---------------|
| **杠杆率** | 当前总仓位 / 账户权益 | 警告 > 2×（黄），危险 > 4×（红） |
| **单笔最大亏损** | 当前未平仓单的最大浮亏 | 警告 > 2%，危险 > 5% |
| **95% VaR** | 95% 置信度下的单日最大亏损 | 警告 > 3%，危险 > 5% |
| **最大回撤** | 当前权益从峰值回撤百分比 | 警告 > 10%，危险 > 20% |

### 9.2 持仓风险矩阵

表格：行 = 标的，列 = 风险等级（低 / 中 / 高 / 极高）

- 单元格颜色：绿 → 黄 → 橙 → 红
- 点击单元格查看该标的详细风险分析

### 9.3 止损止盈状态

3 个列表：

| 状态 | 说明 |
|------|------|
| **已触发止损** | 已平仓，记录盈亏 |
| **待触发止损** | 当前价格距离止损价的百分比（实时监控） |
| **追踪中** | 追踪止损的当前最高价（做多）/ 最低价（做空） |

### 9.4 风险预警

实时从 WebSocket 接收：

| 级别 | 图标 | 说明 |
|------|------|------|
| 警告 | 🟡 | 接近止损线 / 回撤 > 8% |
| 危险 | 🔴 | 触发止损 / 回撤 > 15% / VaR 超限 |
| 紧急 | 🔴 **闪烁** | 回撤 > 20% / 连续 5 笔亏损 |

## 10. 知识图谱

### 10.1 知识图谱可视化

D3.js v7 力导向图，展示鲁兆理论的 **概念关系网络**：

- **节点**：鲁兆理论概念（7 个理论 + DNA29/DNA13 + 太极中心 + ...）
- **边**：关系（派生 / 对立 / 相关）
- **节点大小**：该概念在历史信号中的出现频率
- **节点颜色**：对应理论的主题色

**交互**：
- 鼠标悬停：高亮相邻节点和边
- 鼠标拖动：固定/释放节点
- 滚轮缩放：放大/缩小
- 双击节点：打开节点详情抽屉

### 10.2 DNA 时间窗日历

标注未来 **DNA29 / DNA13** 时间窗口：


## 12. 回测引擎详解

### 12.1 事件驱动架构

回测引擎的核心是 **事件循环**，对每一根历史 K 线依次执行：

```
对于每根 K 线（按时间顺序）：
  1. 标记持仓市值（mark_to_market）
  2. 运行 7 个理论引擎（并行）
  3. 信号融合（FusionStrategy）
  4. 生成订单（Order）
  5. 撮合（OrderBook.match）
  6. 更新持仓（Portfolio.update）
  7. 记录权益曲线
  8. 广播进度（每 100 根 K 线）
```

### 12.2 信号融合策略

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| **AND** | 所有启用的理论一致发出才交易 | 高置信度，低频率 |
| **OR** | 任一理论发出即交易 | 低置信度，高频率 |
| **WEIGHTED** | 按理论权重和置信度加权平均 | 平衡（推荐） |

### 12.3 仓位管理

| 方法 | 说明 | 公式 |
|------|------|------|
| `fixed_pct` | 固定百分比 | `仓位 = 总资金 × 风险比例` |
| `fixed_amount` | 固定金额 | `仓位 = 固定数量` |
| `kelly` | Kelly 公式 | `仓位 = 胜率 - 败率/赔率` |
| `risk_parity` | 风险平价 | 根据各理论的风险贡献分配 |

### 12.4 绩效指标（16 项）

| 指标 | 公式 / 说明 |
|------|----------------|
| 总收益率 | `(期末权益 - 初始资金) / 初始资金` |
| 年化收益率 | `(1 + 总收益率)^(252/交易日数) - 1` |
| 夏普比率 | `平均收益率 / 收益率标准差 × √252` |
| 索提诺比率 | `平均收益率 / 下行偏差 × √252` |
| 卡尔玛比率 | `年化收益率 / |最大回撤|` |
| 最大回撤 | `max((峰值 - 当前) / 峰值)` |
| 胜率 | `盈利交易数 / 总交易数` |
| 盈亏比 | `平均盈利 / |平均亏损|` |
| VaR (95%) | 5% 分位数的亏损值 |
| CVaR (95%) | 超过 VaR 的平均亏损 |

## 13. 参数扫描

### 13.1 配置参数扫描

在回测配置表单底部展开 **「高级 → 参数扫描」**：

1. 点击 **「添加参数范围」**
2. 选择要扫描的参数（如 `theory_weights.DNA29`、`stop_loss_pct`）
3. 设置取值范围（如 `[0.1, 0.2, 0.3, 0.5]`）
4. 选择优化目标（`sharpe_ratio` / `max_drawdown` / `total_return`）
5. 点击 **「启动参数扫描」**

### 13.2 并行执行

系统将参数组合分发给 **Celery Worker 池**（默认 4 并发）：

```
参数组合总数 = Π(各参数的取值数)
示例：[0.1, 0.2, 0.5] × [0.03, 0.05] = 6 组
```

### 13.3 查看扫描结果

完成后展示 **参数排名表**：

| 排名 | 参数组合 | 夏普比率 | 最大回撤 | 总收益率 |
|------|----------|----------|----------|----------|
| 1 | `theory_weights={...}, stop_loss=0.05` | 2.31 | -6.2% | +31.5% |
| 2 | ... | ... | ... | ... |

点击任一行可 **查看该组参数的详细回测结果**。

## 16. 常见问题（FAQ）

### Q1：回测引擎支持哪些市场？

**A**：目前支持 **币安（Binance）** 和 **A 股**。A 股数据通过通达信接口（pytdx）获取，需要配置行情服务器 IP。

### Q2：回测速度如何？

**A**：5 年日线数据（~1500 根 K 线）通常在 **15 秒内**完成。使用 numpy 向量化计算指标，避免 Python 循环。

### Q3：参数扫描会运行多久？

**A**：取决于参数组合数和 Worker 数量。100 组参数在 4 Worker 并行下约 **6 分钟**。

### Q4：如何激活 TOMAS 作家引擎？

**A**：在 `.env` 中配置 `DEEPSEEK_API_KEY`（或 `OPENAI_API_KEY`）。未配置时系统自动降级为 Mock 模式（使用理论引擎信号）。

### Q5：A 股自动交易合规吗？

**A**：当前版本 A 股自动下单为 **Mock 模式**，不会真实下单。真实 A 股自动交易需要：
1. 有合规的量化交易资质
2. 对接合规的券商 API
3. Artificial Immune System（AIS）审核策略

建议在 **纸交易模式** 下充分验证策略后再考虑真实交易。

### Q6：的红涨绿跌可以改吗？

**A**：可以。在 **设置 → 系统** 中切换「涨跌颜色方案」（红涨绿跌 / 绿涨红跌）。

### Q7：如何备份我的布局和偏好设置？

**A**：在 **设置 → 系统** 中点击「导出配置」，将下载一个 JSON 文件。在新设备上点击「导入配置」上传该文件即可恢复。

### Q8：回测结果可以对比吗？

**A**：可以。在回测历史列表中选择 **2-4 个回测记录**，点击「对比」，系统将并排展示权益曲线和绩效指标。
